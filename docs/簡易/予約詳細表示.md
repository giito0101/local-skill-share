# 予約詳細表示

## 概要
提供者が予約内容を確認し、チャット開始・承認・キャンセル操作を行う。

## 対象画面
- `/reservations/[id]`

## API
- `app/reservations/[id]/page.tsx`
- `updateReservationStatusAction`
- `startConversationAction`

## 使用テーブル
- `Reservation`
- `Skill`
- `Conversation`
- `User`

## 入力
| 項目 | 型 | 必須 |
|---|---|---|
| id | string | ○ |
| reservationId | string | ○ |
| intent | string | ○ |

## 出力
- 予約日時
- スキル名
- メッセージ
- ステータス
- チャットボタン
- 承認/キャンセルボタン

## バリデーション
- ログイン必須
- 予約ID必須
- 操作は `approve` / `cancel` のみ

## 異常系
| 内容 |
|---|
| 予約不存在 |
| スキル所有者でない |
| 処理済み予約 |
| 過去予約キャンセル |
| エラーまたは `notFound()` |

## シーケンス
提供者 → 予約詳細 → 認証/権限確認 → 詳細表示 → 必要に応じて状態更新/会話作成

## 備考
- 承認/キャンセルは未来かつ `PENDING` の予約のみ表示
